# LLM Model Probe — UI Design

**Date**: 2026-05-01
**Status**: approved (verbal)
**Builds on**: `docs/specs/2026-05-01-design.md` (the CLI/storage spec is unchanged)

## Goal

Add a local management web UI on top of the existing CLI/SQLite tool so the user
can register and inspect endpoints via copy-paste rather than typing CLI flags.

Primary UX wins over CLI:

- **Smart paste**: drop a JSON snippet, dotenv block, or curl command into one
  textarea and have the form auto-fill.
- **At-a-glance status**: a sortable table of all endpoints with current ok/fail
  counts, no remembering names or running multiple `probe show` calls.
- **One-click retest / delete** per row.

## Non-Goals

- Authentication, multi-user, or remote/public deployment. UI is local-only.
- Live streaming retest progress (Server-Sent Events). v1 blocks the request and
  refreshes the row when done.
- Editing `~/.llm-model-probe/config.toml` from the UI (read-only display).
- Bulk-paste import (paste a JSON array of endpoints at once).
- In-place edit of an endpoint (still `delete` + `add`, same as CLI).
- Auto-refreshing tables. Manual refresh / triggered refresh after actions only.

## Architecture

```
React SPA (Vite + TS + Tailwind + shadcn/ui)
        │   fetch  /api/*
        ▼
FastAPI app (src/llm_model_probe/api.py)
        │   direct calls (no extra layer)
        ▼
Existing Python: store.py · probe.py · settings.py · models.py
        │
        ▼
~/.llm-model-probe/probes.db   (also writable by CLI; same file)
```

**Dev workflow**: two processes — `uvicorn` on 8765, `vite` on 5173. Vite proxies
`/api/*` to 8765. The CLI command `probe ui --dev` documents this; in practice
the user runs `cd frontend && npm run dev` in one shell and `probe ui --dev` in
another.

**Prod / Docker workflow**: single process. `npm run build` produces
`frontend/dist/`; FastAPI mounts that as static files at `/`. `probe ui` (no
flag) starts uvicorn and opens the browser.

## REST API

All routes prefixed `/api`. Responses are JSON. `api_key` is **never** returned
in full — list responses omit it; detail returns `mask_api_key()` form.

| Method | Path                                | Purpose |
|--------|-------------------------------------|---------|
| GET    | `/api/health`                       | `{"ok": true}` for liveness |
| GET    | `/api/endpoints`                    | List w/ summary `{ok, fail, last_tested_at}` |
| GET    | `/api/endpoints/{name_or_id}`       | Detail incl. `results: ModelResultPublic[]` |
| POST   | `/api/endpoints`                    | Create + probe (unless `no_probe=true`) |
| DELETE | `/api/endpoints/{id}`               | Delete (cascade `model_results`) |
| POST   | `/api/endpoints/{id}/retest`        | Single retest, returns updated detail |
| POST   | `/api/retest-all`                   | Bulk retest (blocks; ~30s for ≤20 endpoints) |
| POST   | `/api/parse-paste`                  | `{blob: string}` → `{suggested: Partial<EndpointForm>, confidence}` |
| GET    | `/api/settings`                     | Read-only view of `Settings` |

### Pydantic schemas (api-side)

```python
class EndpointSummary(BaseModel):
    id: str
    name: str
    sdk: Literal["openai", "anthropic"]
    base_url: str
    mode: Literal["discover", "specified"]
    note: str
    list_error: str | None
    available: int          # ok count
    failed: int             # fail count
    last_tested_at: datetime | None
    created_at: datetime
    updated_at: datetime

class ModelResultPublic(BaseModel):
    model_id: str
    source: Literal["discovered", "specified"]
    status: Literal["available", "failed"]
    latency_ms: int | None
    error_type: str | None
    error_message: str | None
    response_preview: str | None
    last_tested_at: datetime | None

class EndpointDetail(EndpointSummary):
    api_key_masked: str     # mask_api_key(api_key)
    models: list[str]       # specified-mode list (empty for discover)
    results: list[ModelResultPublic]

class EndpointCreate(BaseModel):
    name: str = Field(min_length=1)
    sdk: Literal["openai", "anthropic"]
    base_url: HttpUrl
    api_key: str = Field(min_length=1)
    models: list[str] = []  # empty → discover mode
    note: str = ""
    no_probe: bool = False

class PasteParseRequest(BaseModel):
    blob: str

class PasteParseResponse(BaseModel):
    suggested: dict          # subset of EndpointCreate fields
    confidence: float        # 0..1, helpful but not displayed
    parser: str              # "json" | "dotenv" | "curl" | "none"
```

### Pages / interactions

There is **one page**. Layout:

```
┌──────────────────────────────────────────────────────────────┐
│ llm-model-probe                              [+ Add]  [↻ All] │
├──────────────────────────────────────────────────────────────┤
│ ID         Name           SDK   Mode      Status   Tested   │
│ ep_a1b2    bob-glm        openai discover 8/12 ✓   2h ago   │
│ ep_c3d4    partner-claude anthr  specified 2/2 ✓   5m ago   │
│ ...                                                          │
└──────────────────────────────────────────────────────────────┘
```

- Each row has hover-revealed `[retest]` and `[delete]` buttons.
- Clicking the row body opens a **right-side Drawer** with model-level details
  (Available / Failed tables, base URL, masked key, note, list error).
- `[+ Add]` opens a **Dialog** with the form + SmartPaste textarea.
- `[↻ All]` opens a confirm-then-block flow: button shows spinner with running
  count, refresh table when done.
- No router. Server state via `@tanstack/react-query` (caching + auto-
  invalidation after mutations). UI state (selected row, dialog open) via local
  `useState` only — no global store.

### SmartPaste

Two-layer parsing:

1. **Client-side first** (`lib/parsePaste.ts`) — try JSON.parse, dotenv-style
   regex, curl regex. Sufficient for clean inputs, instant feedback.
2. **Server fallback** (`POST /api/parse-paste`) — same logic in Python; called
   if client returns `confidence < 0.5`. Useful for noisy / multi-line shell
   blocks where a Python parser is more robust.

Parser priority (first match wins):
- **json**: `JSON.parse` succeeds, object has any of `base_url`, `apiKey`,
  `api_key`, `models`.
- **curl**: contains `curl ` and an URL token. Extract URL → `base_url` (drop
  path beyond `/v1`); extract `Authorization: Bearer <token>` → `api_key`.
- **dotenv/kv**: lines like `^([A-Za-z_][A-Za-z0-9_]*)\s*[:=]\s*(.+)$`. Map any
  key matching `*BASE_URL*` → base_url, `*API_KEY*` or `*KEY*` → api_key.

If `base_url` contains `anthropic` → suggest `sdk=anthropic`, else `openai`.

Parser return: `{ name?, sdk?, base_url?, api_key?, models?: string[], parser, confidence }`.

UX: textarea has placeholder showing all 3 example formats. Below it: "auto-detected" badge naming the parser, with a "Apply" button (disabled if confidence=0). Apply fills the form fields above.

## CLI changes

New command in `cli.py`:

```python
@app.command()
def ui(
    port: int = typer.Option(8765, "--port"),
    no_browser: bool = typer.Option(False, "--no-browser"),
    dev: bool = typer.Option(False, "--dev",
        help="Dev mode: assume vite at :5173, do not serve dist/"),
) -> None:
    """Start the local web UI."""
```

Behavior:
- Resolve `frontend/dist/` next to package; if missing **and** not `--dev`, print
  a clear "build the frontend first: `cd frontend && npm install && npm run
  build`" message and exit 1.
- Run `uvicorn llm_model_probe.api:app --port <port>` (without reload).
- If `--no-browser` not set, `webbrowser.open(f"http://localhost:{port}")`.
- `--dev` sets env `LLM_MODEL_PROBE_DEV=1` so api.py knows not to mount static.

Existing CLI commands unchanged.

## Backend file additions

```
src/llm_model_probe/
├── api.py            # FastAPI app, routers, Pydantic schemas, parse-paste
└── (existing files unchanged except cli.py adds `ui` cmd)
```

`api.py` structure:
- `app = FastAPI()`
- CORS allows `http://localhost:5173` only (dev convenience).
- Routers: `endpoints_router`, `meta_router` (settings + parse-paste + health).
- Static mount conditional on `LLM_MODEL_PROBE_DEV` env.

Reuses existing `EndpointStore` / `ProbeRunner` / `load_settings()` directly —
no abstraction layer.

## Docker

`Dockerfile` (multi-stage, in repo root):

```dockerfile
# Stage 1: build frontend
FROM node:20-alpine AS frontend
WORKDIR /app/frontend
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ ./
RUN npm run build

# Stage 2: backend + static files
FROM python:3.11-slim
RUN pip install --no-cache-dir uv
WORKDIR /app
COPY pyproject.toml uv.lock README.md ./
COPY src/ ./src/
RUN uv sync --frozen --no-dev
COPY --from=frontend /app/frontend/dist ./frontend/dist
ENV LLM_MODEL_PROBE_HOME=/data
EXPOSE 8765
CMD ["uv", "run", "uvicorn", "llm_model_probe.api:app", "--host", "0.0.0.0", "--port", "8765"]
```

`docker-compose.yml`:

```yaml
services:
  probe:
    build: .
    ports:
      - "8765:8765"
    volumes:
      - ~/.llm-model-probe:/data
    restart: unless-stopped
```

Mounting host `~/.llm-model-probe` means the CLI on the host and the UI in the
container share the same DB.

## Error Handling

| Situation | Behavior |
|---|---|
| Frontend build missing on `probe ui` | clear message, exit 1 |
| API key absent on POST | 422 (Pydantic) |
| Endpoint name conflict | 409 with `{detail: "name exists"}` |
| Endpoint not found by id/name | 404 |
| `list_models()` fails on add | 200 with `endpoint.list_error` set |
| `list_models()` fails on retest | 200; prior model_results retained, new `list_error` set |
| Parse-paste fails entirely | 200 with `parser: "none", confidence: 0` |
| Bulk retest takes too long | timeout left to the client; FastAPI default no timeout. UI uses `fetch` w/o timeout — for v1, just block. |

## Testing

- Backend: pytest tests for the API layer using FastAPI's `TestClient`.
  - happy paths for each route
  - parse-paste: each parser branch (json / dotenv / curl / none)
  - api_key never appears in any response payload (regression guard)
- Frontend: skip unit tests for v1. Manual smoke after build (open page, add
  endpoint, retest, delete).

## Security Notes

- Local-only by default (binds to `localhost`/`127.0.0.1`). The Docker image
  binds `0.0.0.0` because that's standard for containers — user is responsible
  for not exposing the port outside their host network.
- API keys never returned to client; CLI mask format used in detail view.
- No CSRF (single-origin local app); CORS allows only the dev origin.

## Out-of-Scope (explicit)

These were considered and deferred:

- Live retest progress streaming (SSE / WebSocket)
- Settings editor UI (config.toml is read-only via API)
- Edit-in-place for endpoints
- Bulk paste / import
- Multi-user / auth
- Login persistence beyond browser session (no sessions exist v1)
- HTTPS termination (Docker user can put a reverse proxy in front)
