# LLM Model Probe — Design

**Date**: 2026-05-01
**Status**: approved (verbal)

## Goal

Receive ad-hoc OpenAI-compatible / Anthropic API key + base_url combos (often forwarded
from third parties), persist them in a local registry, and probe which models are
actually usable. Re-test on demand.

Primary user workflow:

1. Someone hands over a new `(base_url, api_key, [optional models])`.
2. User runs `probe add ... --note "from Bob"` — entry stored, probed immediately.
3. Days later, user runs `probe list` to see the current usable-model picture across
   all keys, or `probe retest --all` to re-verify.

## Non-Goals

- Time-series history (only the latest result per model is kept; new probe overwrites).
- Web UI / dashboard.
- Auto-scheduled probing (cron). Manual `retest` only.
- API key encryption-at-rest. Rely on file permissions (0600) + masked CLI display.
- Multi-user / shared deployment.

## Architecture

```
CLI (Typer) ── dispatches ──▶ command handler
                                    │
                                    ├─▶ EndpointStore   (SQLite read/write)
                                    │
                                    ├─▶ ProbeRunner     (async, Semaphore concurrency)
                                    │       └─▶ Provider (OpenAI | Anthropic)
                                    │
                                    └─▶ Reporter        (CLI table, markdown, JSON)
```

The async client is reused for one endpoint's whole probe run, then closed.

## Data Model (SQLite)

```sql
CREATE TABLE endpoints (
    id          TEXT PRIMARY KEY,         -- short id, e.g. ep_a1b2c3
    name        TEXT NOT NULL UNIQUE,     -- user-given alias
    sdk         TEXT NOT NULL,            -- 'openai' | 'anthropic'
    base_url    TEXT NOT NULL,
    api_key     TEXT NOT NULL,            -- plaintext; file 0600
    mode        TEXT NOT NULL,            -- 'discover' | 'specified'
    models_json TEXT NOT NULL DEFAULT '[]',  -- specified model list (JSON array)
    note        TEXT NOT NULL DEFAULT '',
    list_error  TEXT,                     -- last list_models() error if any
    created_at  TEXT NOT NULL,
    updated_at  TEXT NOT NULL
);

CREATE TABLE model_results (
    endpoint_id      TEXT NOT NULL,
    model_id         TEXT NOT NULL,
    source           TEXT NOT NULL,       -- 'discovered' | 'specified'
    status           TEXT NOT NULL,       -- 'available' | 'failed'
    latency_ms       INTEGER,
    error_type       TEXT,
    error_message    TEXT,
    response_preview TEXT,
    last_tested_at   TEXT NOT NULL,
    PRIMARY KEY (endpoint_id, model_id),
    FOREIGN KEY (endpoint_id) REFERENCES endpoints(id) ON DELETE CASCADE
);

CREATE INDEX idx_model_results_endpoint ON model_results(endpoint_id);
```

Re-probe of an endpoint replaces its `model_results` rows with the new set (per probe
flow rules — see _Probe Semantics_ for `list_models` failure handling). Simple, no
migration, matches "I just want to know what works now".

## CLI Commands

```
probe add --name <alias> --sdk <openai|anthropic>
          --base-url <url> --api-key <key>
          [--models m1,m2,...]      # if omitted: discover mode
          [--note <text>]
          [--no-probe]              # add only, don't probe immediately

probe list [--json]                 # all endpoints with summary line

probe show <name|id> [--json]       # detailed model results

probe retest <name|id> [--force]    # re-probe one endpoint
probe retest --all [--force]        # re-probe all; skip any tested <24h ago
                                    # unless --force

probe rm <name|id> [--yes]          # delete endpoint + its results

probe export [name|id]              # if no arg: aggregate all
             [--format md|json]
             [-o <file>]            # default: stdout
```

`<name|id>` accepts either the user-given alias or the short id.

`probe list` output (rich-rendered table):

```
ID       NAME           SDK        AVAIL   FAIL   TESTED      NOTE
ep_a1b2  bob-glm        openai     8/12     4    2h ago      Bob 5/1 给的
ep_c3d4  zhipu-trial    openai     0/0    list-error 5d ago  ...
```

## Probe Semantics

**Discovery flow** (mode=discover):
1. `client.models.list()` → list of model IDs.
2. Apply global exclude filter from `config.toml`.
3. Probe each kept model concurrently (asyncio.Semaphore).

**Specified flow** (mode=specified):
1. Skip listing, skip global filter.
2. Probe each user-supplied model directly.

**Probe call** (per model):
- OpenAI: `chat.completions.create(model=..., messages=[{"role":"user","content":prompt}], max_tokens=8)`
- Anthropic: `messages.create(model=..., max_tokens=8, messages=[...])`
- On `max_completion_tokens` error (reasoning models): retry with that param instead.
- Capture: latency_ms, `error_type` (exception class name), `error_message` (truncated 300 chars),
  `response_preview` (first 80 chars of completion).

**list_models() failure**:
- On initial `add` (discover mode): record `list_error` on endpoint, no `model_results` rows. Endpoint shows in `probe list` as `list-error`.
- On `retest` (discover mode): record new `list_error`, **keep** prior `model_results` rows
  (their `last_tested_at` reflects when they were last probed). Surfaces "list broke, but
  here's the most recent picture we had" rather than blanking everything.
- Recovery: `probe rm <name>` then re-`add`, or use specified mode by re-adding with
  `--models`. Explicit `edit` command is out of scope.

## Storage Layout

| Path | Purpose | Perms |
|---|---|---|
| `~/.llm-model-probe/probes.db` | SQLite database | 0600 |
| `~/.llm-model-probe/config.toml` | global probe + filter settings | 0600 |
| Directory itself | container | 0700 |

Override base via env `LLM_MODEL_PROBE_HOME=/some/path`.

First run creates the directory + a default `config.toml` if missing.

## Configuration (`config.toml`)

```toml
[probe]
concurrency = 5
timeout_seconds = 30
max_tokens = 8
prompt = "Hi"
retest_cooldown_hours = 24       # used by `retest --all`

[filters]
exclude = [
    "*embedding*", "*embed*",
    "*whisper*", "*tts*", "*audio*",
    "*dall-e*", "*image*",
    "*moderation*", "*rerank*",
]
```

## Error Handling

| Situation | Behavior |
|---|---|
| Endpoint name conflict on `add` | Reject with clear message |
| `--name` or `--id` not found | Reject with clear message |
| `list_models()` raises | Persist as `list_error`, report to user, continue |
| Per-model probe raises | Persist as `status=failed` with error details |
| SQLite locked | One short retry, then surface error |
| Bad CLI args | Typer's default validation |
| Missing config.toml | Auto-create with defaults |

## Testing

- Unit:
  - filter pattern matching (`probe._filter_models`)
  - config.toml loader defaults + overrides
  - SQLite store: insert/upsert/replace cycle for endpoints + results
  - id generation uniqueness
- Manual smoke:
  - `uv run probe --help`
  - `uv run probe add ...` against a real (or mock) endpoint
  - `uv run probe list / show / retest / export` round-trip
